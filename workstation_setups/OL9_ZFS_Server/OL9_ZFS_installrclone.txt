# ‚úÖ 3Ô∏è‚É£ Configure rclone for Google Drive

Run configuration as the rclone user:

```bash
sudo -u rclone rclone config
```

### Interactive Steps

1. `n` ‚Üí New remote
2. Name it: `gdrive`
3. Storage type ‚Üí type `drive`
4. Client ID ‚Üí **leave blank** (or use custom if you have one)
5. Client Secret ‚Üí **leave blank**
6. Scope ‚Üí choose:

   * `1` (full access) usually fine
7. Root folder ID ‚Üí blank
8. Service account file ‚Üí blank (unless using Google Workspace SA)
9. Edit advanced config ‚Üí `n`
10. Auto config ‚Üí `y`
11. not a shared team drive

You will get a URL ‚Üí open it in a browser ‚Üí login ‚Üí copy verification code back.

After completion, config file will be stored at:

```
/etc/rclone/rclone.conf
```

Verify permissions:

```bash
sudo chmod 600 /etc/rclone/rclone.conf
sudo chown rclone:rclone /etc/rclone/rclone.conf
```

# ‚úÖ 6Ô∏è‚É£ Optional: Create Systemd Service (Recommended for Production)

Create file:

```bash
sudo nano /etc/systemd/system/rclone-gdrive.service
```

Paste:

```ini
[Unit]
Description=Rclone Google Drive Mount
After=network-online.target

[Service]
Type=simple
User=rclone
Group=rclone
ExecStart=/usr/bin/rclone mount gdrive: /mnt/gdrive \
  --config=/etc/rclone/rclone.conf \
  --allow-other \
  --dir-cache-time=72h \
  --vfs-cache-mode=writes
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

Enable + start:

```bash
sudo systemctl daemon-reload
sudo systemctl enable rclone-gdrive
sudo systemctl start rclone-gdrive
```

Check status:

```bash
systemctl status rclone-gdrive
```
